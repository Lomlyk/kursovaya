## АННОТАЦИЯ

Настоящий курсовой проект выполнен по дисциплине «Объектно-ориентированное программирование Python». В рамках проекта разработано настольное приложение `Math Modules Explorer`, демонстрирующее практическое применение модулей Python для работы с математическими функциями, статистическими вычислениями, точной арифметикой и визуализацией результатов.

В проекте использованы основные принципы объектно-ориентированного программирования. Инкапсуляция обеспечена разделением интерфейса и вычислительной логики по отдельным классам. Абстракция реализована через специализированные сервисные классы, каждый из которых отвечает за собственную задачу: вычисление выражений, решение квадратных уравнений, расчет статистики, финансовые вычисления и подготовку данных для построения графика. Полиморфизм проявляется в едином способе вызова вычислительных сервисов, а модульность повышает читаемость и расширяемость программы.

Для обеспечения безопасности приложение использует разбор математических выражений через модуль `ast`, что позволяет запретить выполнение постороннего кода. Графический интерфейс реализован на базе стандартной библиотеки `tkinter`. Корректность логики подтверждена набором модульных тестов.

## 3 Разработка

### 3.1 Архитектура проекта

```text
kursovaya_math_modules/
├── main.py
├── math_engine.py
├── requirements.txt
├── docs/
│   ├── course_report.md
│   ├── defense_notes.md
│   └── test_cases.md
└── tests/
    └── test_math_engine.py
```

Приложение разделено на несколько частей:

- `main.py` содержит графический интерфейс пользователя;
- `math_engine.py` содержит вычислительную логику проекта;
- `tests/` содержит модульные тесты;
- `docs/` содержит пояснительную записку, материал для защиты и сценарии тестирования.

### 3.2 Реализация принципов ООП

#### Инкапсуляция

Вычислительная логика не смешивается с графическим интерфейсом. Все математические операции вынесены в отдельные классы:

- `ExpressionEvaluator`
- `QuadraticSolver`
- `StatisticsService`
- `FinanceMath`
- `FunctionSampler`

Такой подход упрощает сопровождение проекта и делает код удобным для тестирования.

#### Абстракция

Каждый класс предоставляет только тот интерфейс, который нужен для решения одной конкретной задачи. Например, `QuadraticSolver` отвечает только за квадратные уравнения, а `FinanceMath` только за финансовые вычисления.

#### Полиморфизм и модульность

Сервисы используются в единой логике приложения через вызов их методов из GUI-слоя. Это позволяет без существенных изменений расширять проект новыми вычислительными блоками.

### 3.3 Обработка исключений

Приложение обрабатывает типовые ошибочные ситуации:

- пустой ввод;
- недопустимые функции и идентификаторы;
- деление на ноль;
- неправильные коэффициенты квадратного уравнения;
- некорректные числовые данные;
- ошибочные границы интервала.

Для безопасного вычисления выражений используется разбор через `ast`, а затем разрешается только ограниченный набор узлов и математических функций.

### 3.4 Работа с модулями Python

В проекте использованы следующие модули стандартной библиотеки:

| Модуль | Назначение |
|---|---|
| `math` | Тригонометрические, степенные, логарифмические функции |
| `statistics` | Среднее значение, медиана |
| `decimal` | Точные финансовые расчеты |
| `ast` | Безопасный анализ пользовательского выражения |
| `tkinter` | Графический интерфейс |

### 3.5 Графический интерфейс

Интерфейс реализован на базе `tkinter` и состоит из пяти вкладок:

| Вкладка | Описание |
|---|---|
| Calculator | Вычисление математических выражений |
| Quadratic | Решение квадратных уравнений |
| Statistics | Расчет статистических показателей |
| Decimal | Расчет сложных процентов |
| Plot | Построение графика функции |

## 4 Тестирование

Для проверки корректности проекта разработаны модульные тесты.
